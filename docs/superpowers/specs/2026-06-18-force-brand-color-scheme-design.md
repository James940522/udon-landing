# 브랜드 컬러 스킴 고정 설계

## 목적

시스템이 다크 모드여도 삼성 인터넷을 포함한 브라우저가 페이지 색상을 자동으로 어둡게 변환하지 않도록 한다. 사이트는 현재 정의된 프리미엄 이자카야 브랜드 팔레트를 항상 그대로 표시한다.

## 선택한 방식

HTML 문서의 초기 메타데이터와 전역 CSS에 라이트 전용 컬러 스킴을 함께 선언한다.

- Next.js 메타데이터에 `color-scheme: light`를 선언해 CSS가 로드되기 전부터 브라우저에 페이지의 지원 컬러 스킴을 알린다.
- 전역 `:root`에 `color-scheme: only light`를 선언해 브라우저의 자동 다크 컬러 재정의를 거부한다.
- `html`과 `body`의 배경색을 기존 브랜드 배경 변수로 명시해 초기 렌더링과 페이지 바깥 캔버스의 색상도 일관되게 유지한다.

`only light`는 표준 CSS Color Adjustment 구문이며, 브라우저 자동 다크 테마의 색상 재정의를 차단하기 위한 선언이다.

## 검토한 대안

### CSS 선언만 사용

`color-scheme: only light`만 적용하는 가장 작은 변경이다. 자동 다크 변환은 막을 수 있지만 CSS가 적용되기 전 초기 화면에서 브라우저 기본 색상이 잠깐 노출될 수 있어 선택하지 않는다.

### 삼성 인터넷 전용 감지 스크립트

사용자 에이전트를 감지해 삼성 인터넷에서 별도 클래스를 적용할 수 있다. 브라우저 버전과 사용자 에이전트 변화에 취약하고 다른 Chromium 계열의 자동 다크 모드에는 대응하지 못하므로 선택하지 않는다.

### 메타데이터와 CSS 이중 방어

초기 렌더링과 실제 스타일 계산 양쪽을 모두 제어한다. 표준 기능만 사용하고 삼성 인터넷 외 브라우저에서도 시스템 테마와 무관하게 동일한 브랜드 색상을 유지하므로 이 방식을 선택한다.

## 변경 범위

- `src/app/layout.tsx`
  - 색상 스킴 메타데이터 추가
- `src/app/globals.css`
  - 전역 `only light` 컬러 스킴 선언
  - 문서와 본문 배경색 명시

기존 `.dark` 클래스 토큰과 개별 컴포넌트 색상은 변경하지 않는다. 사이트가 향후 명시적인 다크 테마를 제공하게 되면 이번 고정 선언을 제거하거나 해당 테마 구조에 맞게 조정한다.

## 검증

- 생성된 HTML에 `meta[name="color-scheme"][content="light"]`가 포함되는지 확인한다.
- 전역 CSS에 `:root { color-scheme: only light; }`가 포함되는지 확인한다.
- 프로덕션 빌드와 린트를 실행해 Next.js 메타데이터 타입 및 CSS 빌드 오류가 없는지 확인한다.
- 가능하면 Chromium의 강제 자동 다크 모드 환경에서 브랜드 배경색과 텍스트색이 유지되는지 확인한다.

## 성공 기준

- 운영체제가 다크 모드여도 사이트 팔레트가 자동 변환되지 않는다.
- 삼성 인터넷에서 기존 브랜드 배경, 텍스트, 버튼, 폼 색상이 유지된다.
- 브라우저 기본 폼 컨트롤과 스크롤바도 라이트 컬러 스킴을 기준으로 렌더링된다.
- 기존 페이지 기능과 반응형 레이아웃에 변화가 없다.
