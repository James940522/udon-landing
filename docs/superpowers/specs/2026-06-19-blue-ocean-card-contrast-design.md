# 블루오션 혜택 카드 대비 강화 설계

## 목표

블루오션 섹션의 밝은 배경과 웜 에디토리얼 톤은 유지하면서, 세 개의 핵심 혜택 카드에
`성공으로 가는 6단계 로드맵`과 같은 또렷한 색상 위계를 적용한다.

로드맵 카드의 디자인 문법을 공유하되, 블루오션 카드의 기존 문구와 3열 구조는 유지한다.

## 확정 디자인

### 카드 표면

- 카드 배경은 현재 `bg-[#fff8eb]/88`보다 또렷한 `bg-[#fffaf2]/94`로 변경한다.
- 테두리는 골드 중심의 `border-[#c9a24d]/38` 대신 로드맵과 같은 눌린 베이지
  `border-[#d8c8b5]/90`을 사용한다.
- 카드 상단에는 높이 `4px`의 적갈색 `#9b5b46` 바를 추가한다.
- 그림자는 브라운 계열로 유지하되 카드 외곽이 배경과 확실히 분리되도록 농도를 조금 높인다.

### 번호와 배지

- 현재 흐릿한 대형 `01`, `02`, `03` 텍스트는 제거한다.
- 좌측 상단에는 적갈색 `#9b5b46` 바탕과 아이보리 글자의 원형 번호 배지를 배치한다.
- 번호 배지는 아이보리 외곽선과 브라운 그림자를 사용해 배경과 겹쳐도 또렷하게 보이게 한다.
- 우측 하이라이트 배지는 크림 바탕, 적갈색 텍스트와 베이지 테두리를 유지하되 카드와 구분되는
  불투명도로 조정한다.

### 카드 본문과 동작

- 카드 내용은 번호와 하이라이트 배지 아래에서 시작하도록 상단 여백을 확보한다.
- 제목은 짙은 브라운 `#2b1b16`, 설명은 중간 브라운 `#725744`를 유지한다.
- hover 시 카드는 가볍게 상승하고 그림자만 적갈색 방향으로 조금 깊어진다.
- 강한 글로우나 새로운 이미지 장식은 추가하지 않는다.

## 비변경 범위

- 블루오션 배경 이미지, 배경 베일, 헤드라인, 문제 제기 영역은 변경하지 않는다.
- 인용구 카드와 최종 메시지는 변경하지 않는다.
- 혜택 문구, 배열, 카드 순서, 3열 반응형 구조와 애니메이션 타이밍은 유지한다.
- 외부 패키지와 새 이미지 에셋은 추가하지 않는다.

## 검증

- 기존 블루오션 계약 테스트에 상단 적갈색 바, 원형 번호 배지, 아이보리 카드와 베이지
  테두리 토큰을 추가한다.
- 기존 흐릿한 대형 번호 스타일과 골드 카드 테두리가 제거됐는지 확인한다.
- 계약 테스트가 먼저 실패한 뒤 구현 후 통과하는지 확인한다.
- 집중 ESLint, 전체 Node 테스트와 프로덕션 빌드를 실행한다.
