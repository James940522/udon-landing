# 블루오션 섹션 웜 에디토리얼 톤 설계

## 목표

`BlueOceanAdvantageSection`의 메시지와 정보 구조는 유지하면서, 현재의 검은 캠페인 배너 같은
인상을 줄이고 앞뒤 섹션에 사용된 크림, 적갈색, 골드 계열과 자연스럽게 연결한다.

섹션 전체를 평평하게 밝히지 않고 문제 제기, 해결책, 핵심 인용구의 명암 위계는 유지한다.

## 확정 디자인

### 배경과 섹션 연결

- 기존 배경 이미지는 유지하되 `bg-black/75` 중심의 다크 오버레이를 제거한다.
- 이미지 위에는 따뜻한 아이보리와 베이지 계열의 반투명 베일을 적용한다.
- 상단은 앞의 `BrandIntroSection`과 이어지는 크림 톤으로 시작한다.
- 하단은 브라운 농도를 조금 높여 다음 섹션으로 넘어가는 시각적 완충 구간을 만든다.
- 배경 이미지가 콘텐츠 가독성을 방해하지 않도록 채도와 대비를 낮춘다.

### 헤드라인

- 기본 제목은 흰색에서 짙은 초콜릿 브라운으로 변경한다.
- `현시점 배달업계 최대의 블루오션`은 브랜드 골드로 강조한다.
- `'볶음우동'`은 브랜드 적갈색 `#8f3528`로 강조해 골드 강조가 연속되는 문제를 줄인다.
- 상단 경고 배지는 선명한 빨강 대신 탁한 테라코타 바탕과 크림색 텍스트를 사용한다.
- 부제는 붉은 경고색 대신 적갈색 계열로 정리한다.

### 문제 제기 영역

- 외부 패널은 눌린 로즈 베이지 `#e5c8b5` 계열의 반투명 배경으로 변경한다.
- 세 개의 문제 카드는 밝은 아이보리 카드로 구성하고 브라운 테두리와 낮은 그림자로
  분리한다.
- 문제명과 결과는 적갈색, 설명은 중간 브라운을 사용한다.
- 기존 문구, 카드 순서, 3열 구조와 애니메이션은 유지한다.

### 해결책 카드

- 해결책 제목과 본문은 짙은 브라운 계열로 변경한다.
- 세 개의 혜택 카드는 따뜻한 아이보리 바탕, 골드 테두리, 부드러운 브라운 그림자를 사용한다.
- 번호는 낮은 불투명도의 골드로 남기고 하이라이트 배지는 크림·골드 조합으로 정리한다.
- hover는 강한 불꽃 글로우 대신 가벼운 상승, 테두리 강조, 따뜻한 그림자 정도로 제한한다.

### 인용구와 최종 메시지

- `경쟁의 해답은 ... '겹치지 않는 것'` 인용구는 섹션 안의 유일한 짙은 브라운 카드로 남긴다.
- 인용구 카드의 텍스트는 크림색, 핵심 문구는 골드로 유지해 시각적 중심을 만든다.
- 최종 메시지는 밝은 웜 베이지 패널로 바꾸고 적갈색과 골드로 핵심 문장을 강조한다.

## 반응형과 접근성

- 모바일의 기존 문장 줄바꿈과 단일 열 카드 흐름은 유지한다.
- 밝아진 카드와 텍스트 간 명도 대비를 확인한다.
- 배경 이미지와 장식 오버레이는 계속 장식 요소로 처리한다.
- 새로운 이미지 에셋이나 외부 패키지는 추가하지 않는다.

## 변경 범위

- 주요 변경 파일은
  `src/widgets/blue-ocean-advantage/ui/BlueOceanAdvantageSection.tsx`로 한정한다.
- 문구, 데이터 배열, 섹션 순서, 애니메이션 타이밍은 변경하지 않는다.
- `BrandIntroSection`, `TrustedFranchiseSection` 등 인접 섹션은 수정하지 않는다.

## 검증

- 소스 계약 테스트를 추가해 검은 오버레이와 기존 스톤 다크 카드 토큰이 제거됐는지 확인한다.
- 웜 크림 배경, 브라운 텍스트, 밝은 카드, 짙은 인용구 카드가 존재하는지 확인한다.
- 데스크톱과 모바일에서 카드 구조와 줄바꿈이 유지되는지 확인한다.
- ESLint와 프로덕션 빌드를 실행한다.
