# 모바일 성공 로드맵 등장 수정

## 문제

모바일에서 “성공으로 가는 6단계 로드맵” 카드가 보이지 않는다. 여섯 카드가 한 열로 쌓여
그리드 높이가 뷰포트보다 길어지지만, 현재 애니메이션은 전체 그리드의 35%가 동시에 보여야
시작한다. 390×844 뷰포트에서 가능한 최대 노출 비율은 약 31%라 조건을 충족할 수 없다.

## 검토한 접근

1. 모바일에서 카드 애니메이션을 완전히 제거한다.
2. 화면 크기에 따라 교차 비율을 다르게 계산한다.
3. 전체 그리드의 비율 대신 로드맵 영역이 화면에 진입했는지를 기준으로 카드를 활성화한다.

세 번째 방식을 적용한다. 별도 반응형 상태나 자바스크립트 미디어 쿼리를 추가하지 않으면서
모바일과 데스크톱 모두에서 동일한 등장 규칙을 사용할 수 있다.

## 변경

- 카드 활성 조건을 도달 불가능한 `amount: 0.35` 그리드 감지에서 섹션 진입 감지로 통합한다.
- 기존 카드별 지연, 이동, 축소 애니메이션과 모션 감소 설정은 유지한다.
- 모바일에서 다시 같은 임계치가 도입되지 않도록 검증 스크립트에 계약을 추가한다.

## 성공 기준

- 모바일에서 로드맵 제목 아래 여섯 카드가 순차적으로 나타난다.
- 카드는 초기 투명 상태에 영구적으로 머물지 않는다.
- 데스크톱 카드 배치와 순차 등장 효과는 유지된다.
- 기존 로드맵 디자인 계약, 린트, 프로덕션 빌드가 통과한다.
