# 전국 오픈 리포트 매장 수 정합성 설계

## 목표

전국 오픈 리포트의 `TOTAL STORES` 숫자가 화면에 표시되는 실제 유효 매장 목록과 항상 일치하도록 한다.

## 현재 문제

모달은 매장 목록을 `/api/stores`에서 가져오면서, 총 매장 수는 별도 `/api/stores/count`에서 가져온다. 두 API의 파싱 기준과 캐시가 달라 목록과 숫자가 어긋날 수 있다. 실제 CSV의 유효 매장은 23개지만 별도 카운트 응답 때문에 잘못된 숫자가 우선 표시될 수 있다.

## 설계

- `AchievementModal`은 `fetchStores()`가 반환한 유효 매장 배열을 유일한 데이터 기준으로 사용한다.
- `TOTAL STORES`에는 `stores.length`를 표시한다.
- 별도 카운트 훅과 API 응답은 모달 표시값 결정에 사용하지 않는다.
- 목록 로딩 전에는 기존처럼 `-`를 표시하고, 로딩 후에는 목록과 동일한 개수를 표시한다.

## 오류 처리

매장 목록 요청이 실패하면 `fetchStores()`가 빈 배열을 반환한다. 이 경우 기존 빈 상태 UI를 유지하며 잘못된 외부 카운트 값을 대신 표시하지 않는다.

## 검증

- 정적 회귀 테스트에서 모달이 `useStoreCount`를 사용하지 않는지 확인한다.
- 정적 회귀 테스트에서 표시값이 `stores.length`로 계산되는지 확인한다.
- 기존 오픈 리포트 계약 테스트, ESLint, 프로덕션 빌드를 실행한다.
