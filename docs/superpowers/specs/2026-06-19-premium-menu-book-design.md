# 프리미엄 메뉴북 섹션 설계

## 목표

메뉴 섹션의 배경 사진과 무거운 다크 UI를 제거하고, 사이트의 크림·베이지 팔레트와
어울리는 정돈된 프리미엄 메뉴북으로 재구성한다.

## 시각 방향

- 섹션 배경은 따뜻한 아이보리 `#f7f0e4`를 사용한다.
- 카드 배경은 `#fffaf2`, 주요 글자는 `#2b1b16`, 보조 글자는 `#746054`를 사용한다.
- 적갈색 `#9b5b46`은 활성 탭과 작은 인덱스에만 제한적으로 사용한다.
- 배경 이미지, 어두운 오버레이, 큰 브랜드 로고 영역을 제거한다.
- 얇은 선, 넉넉한 여백, 낮은 그림자로 고급감을 만든다.

## 정보 구조

1. 상단에 섹션 번호와 `SIGNATURE MENU` 라벨을 둔다.
2. 제목과 소개 문구를 좌우로 나누어 배치한다.
3. 브랜드 선택 영역은 단일 브랜드이므로 제거한다.
4. 모든 카테고리를 한 줄 탭으로 배치한다. 모바일에서는 가로 스크롤을 허용한다.
5. 선택된 카테고리명과 메뉴 개수를 그리드 상단에 표시한다.
6. 메뉴 카드는 모바일 2열, 태블릿 3열, 넓은 화면 4열로 배치한다.

## 메뉴 카드

- 음식 사진은 정사각 비율을 유지한다.
- 카드는 얇은 베이지 테두리와 아주 낮은 그림자를 사용한다.
- 카드 하단에 두 자리 메뉴 인덱스와 메뉴명을 배치한다.
- hover에서는 확대하지 않고 카드가 4px 정도만 올라오며 테두리가 진해진다.

## 반응형과 상호작용

- 모바일에서도 메뉴는 2열을 유지하되 글자와 여백을 줄인다.
- 카테고리 변경 시 현재의 페이드·슬라이드 전환을 유지한다.
- 메뉴 데이터와 이미지 경로는 변경하지 않는다.
- 단일 브랜드 상태에 불필요한 브랜드 선택 state와 상단 이동 동작은 제거한다.

## 이미지 경로 호환

- `public`의 한글 카테고리 폴더가 분해형 유니코드(NFD)로 저장되어 있으므로 카드 렌더링
  직전에 실제 파일명과 같은 형태로 경로를 정규화한다.
- 파일명까지 NFD인 김치볶음우동, 철판 볶음밥, 사이드와 우삼겹 볶음우동은 파일명도
  함께 정규화한다.
- Next.js 이미지 최적화 프록시가 NFD 경로를 다시 처리하면 400을 반환하므로 메뉴
  카드 이미지는 원본 정적 URL을 직접 사용한다.

## 검증

- 소스 계약 테스트로 배경 이미지 제거, 밝은 팔레트, 탭 구조, 반응형 그리드를 확인한다.
- 경로 정규화 단위 테스트로 NFC/NFD가 섞인 실제 메뉴 자산 구조를 확인한다.
- 메뉴 컴포넌트 ESLint와 프로덕션 빌드를 실행한다.
- 로컬 화면에서 데스크톱과 모바일 레이아웃, 카테고리 전환, 이미지 자연 크기를 확인한다.
